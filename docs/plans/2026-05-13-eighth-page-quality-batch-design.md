# Eighth Page Quality Batch Design

**Problem**

The EBS cluster is already well separated by role, but some visible calculator and support pages can still better reinforce the difference between three distinct storage questions:

- live volume cost
- paid performance headroom
- retained snapshot growth

These pages are not thin. The remaining opportunity is to make them feel more decision-ready and less generic by sharpening the sign-off language and reminding readers which cost surface they are actually modeling.

**Decision**

Run an eighth narrow quality batch focused on the EBS cluster, with small copy improvements that strengthen cost-surface separation and end-of-page validation language.

**Approach**

1. Keep all EBS role-separation language intact.
2. Strengthen page-specific caution wording rather than adding more generic explanatory copy.
3. Make the "live volume vs paid performance vs retained snapshot" distinction even clearer.
4. Avoid structural rewrites and keep edits local to support paragraphs or closing guidance.

**Pages in this batch**

- `src/pages/calculators/aws-ebs-cost-calculator.astro`
- `src/pages/calculators/aws-ebs-snapshot-cost-calculator.astro`
- `src/pages/guides/aws-ebs-pricing.astro`

**Expected outcome**

These EBS pages should feel more original, more trustworthy, and more resistant to "one blended storage total" thinking, while preserving the existing EBS cluster governance.
